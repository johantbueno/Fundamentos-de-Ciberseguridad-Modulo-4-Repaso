La Tríada CIA
Confidencialidad

Propiedad de que la información no se revela a individuos, entidades o procesos no autorizados.

Integridad

Garantía de exactitud y completitud. Los datos no han sido modificados sin autorización (no manipulados).

Disponibilidad

Información accesible y utilizable por una entidad autorizada cuando lo requiera.

Amenazas y Malware
  • Ransomware Malware que cifra archivos y pide rescate económico.
  • Rootkit Software que oculta su presencia en el sistema (nivel kernel).
  • Phishing Ingeniería social para robar credenciales mediante engaño.
  • Botnet Red de equipos infectados controlados remotamente.
  • SQLi Inyección de código en bases de datos a través de webs vulnerables.
Criptografía Básica
Simétrica vs Asimétrica

Simétrica: Usa la misma clave para cifrar y descifrar (Rápida, ej: AES).

Asimétrica: Usa clave Pública (cifrar) y Privada (descifrar). Resuelve el intercambio de claves.

Hash

Función unidireccional que genera un resumen fijo (huella digital). Si cambia un bit del archivo, cambia todo el hash. Vital para verificar Integridad.

Ciclo de Vida (NIST 800-61)
  1. Preparación: Políticas, herramientas, formación del equipo.
  2. Detección y Análisis: Identificar IOCs (Indicadores de Compromiso), triaje inicial.
  3. Contención, Erradicación y Recuperación:
    • Contención: Aislar sistemas para detener el daño.
    • Erradicación: Eliminar la amenaza (borrar malware, cerrar cuentas).
    • Recuperación: Restaurar servicios y validar funcionamiento.
  4. Actividad Post-Incidente: Lecciones aprendidas.
Adquisición de Evidencia
Orden de Volatilidad (RFC 3227)

Se debe recolectar primero lo que se pierde al apagar el sistema:

  1. Memoria RAM, registros de CPU, caché.
  2. Conexiones de red, procesos en ejecución.
  3. Disco Duro (Datos persistentes).
  4. Backups y medios externos.
Hardware Write Blocker

Dispositivo físico fundamental que se conecta entre el disco de evidencia y el equipo forense para impedir cualquier escritura y garantizar la integridad legal de la prueba.

Sistema de Ficheros (NTFS)
  • $MFT (Master File Table): Índice central de todos los archivos. Contiene metadatos (Fechas MACE, permisos).
  • Prefetch (.pf): Rastrea la ejecución de aplicaciones (10 primeros segundos). Ubicado en C:\Windows\Prefetch. Prueba que un programa fue ejecutado.
  • Papelera ($Recycle.Bin): Al borrar, crea dos archivos: $I (Metadatos: fecha, ruta) y $R (Contenido real renombrado).
  • Hiberfil.sys: Copia de la RAM en disco (Hibernación).
Registro de Windows

Base de datos de configuración del sistema (Hives: SYSTEM, SOFTWARE, NTUSER.DAT).

Claves Críticas para Forense:
  • Auto-Run (Persistencia): Claves Run y RunOnce. Donde el malware se configura para iniciar con Windows.
  • ShimCache (AppCompatCache): Rastrea ejecutables para compatibilidad. Prueba la existencia de ficheros pasados.
  • UserAssist: (En HKCU) Monitoriza programas ejecutados vía GUI. Los nombres están ofuscados en ROT13.
  • Shellbags: Rastrea acceso a carpetas y ventanas. Prueba acceso a carpetas específicas.
  • USBStor: Historial de dispositivos USB conectados.
Logs de Eventos

Archivos .evtx. Clave principal:

  • Evento 4624: Inicio de sesión exitoso.
  • Tipos de Logon: Tipo 2 (Teclado/Local), Tipo 3 (Red), Tipo 10 (RDP/Remoto).
El Informe y Remediación
Pautas del Informe
  • Escribir en tiempo pasado (los hechos ya ocurrieron).
  • Evitar jerga técnica en el Resumen Ejecutivo (dirigido a gerencia).
  • Ser preciso, claro y basarse en evidencias verificadas.
Remediación

Acciones para restaurar la seguridad: Cambio global de contraseñas, reconstrucción de sistemas infectados, cierre de vectores de ataque y mejora de monitoreo.
